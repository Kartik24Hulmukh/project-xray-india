# Public Release Trust Record — Specification

Status: specification only; no trust record exists until generated by CI at release time.

Machine-readable JSON published with each release:
- `git_sha` — reviewed release SHA
- `image_digest` — deployed container digest
- `sbom_hash` — SHA-256 of the published SBOM
- `scanner` — tool, version/digest, result summary (counts by severity)
- `cve_exceptions` — list of {id, owner, rationale, expires} (current set expires 2026-08-18)
- `target_receipts` — AWS target verification status per RELEASE_MATURITY_MODEL track 2, or `pending`
- `restore_test` / `rollback_test` — ISO timestamps or `pending`
- `publication_safety_tests` — pass/fail
- `maturity_labels` — subset of the three model labels actually held

Rule: fields must never be omitted to hide a pending state; use explicit `pending`.
