# Founder council and pre-mortem — v0.4.4

This council uses strategic lenses inspired by capital-efficient founders, first-principles engineers, civic-tech operators, editors and maintainers. It does not impersonate or claim endorsement by any real person.

## Verdict

Proceed Monday only as an invitation-only synthetic workflow preview. Do not call it a public launch, corruption detector, verified-truth system, production service, or traction event.

## Highest-probability failure modes

1. Exact anchors create false confidence while context or source selection is wrong.
2. Private fields leak through reports, errors, exports, caches or future schema additions.
3. Gateway authentication is mistaken for application authorization.
4. Two reviewers share the same bias, identity or source-selection error.
5. Kill switches exist but no named operator can activate them quickly.
6. Logical scanner decisions are mistaken for evidence that exact bytes were scanned.
7. Corrections update storage but stale exports/screenshots continue circulating.
8. A partisan actor reframes the system as an accusation or ranking engine.
9. A polished demo is reported as product traction.

## Monday value hypothesis

The defensible product category is **evidence operations**: preserving source versions, linking exact anchors, separating candidate from public material, requiring independent review and maintaining correction/withdrawal history.

Measure workflow time, review reconstruction time, missing-anchor defects, public-canary disclosures and willingness to repeat a second synthetic session. GitHub stars, invitations, compliments and test counts are not traction.

## Stop conditions

Stop immediately for real allegations, personal/confidential data, public-boundary leakage, direct publication bypass, same-person dual review, unverifiable package SHA, inaccessible kill switch, exposed credentials or any unsupported production/impact claim.
