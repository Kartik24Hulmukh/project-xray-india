# Monday 48-hour controlled-preview plan

## T−48 to T−24

- Freeze to one conspicuously fictional procurement scenario.
- Name technical, evidence/editorial, privacy, security/incident and complaint owners.
- Verify package and handoff checksums.
- Disable public uploads, real-case publication and indexing.
- Rehearse candidate → two reviews → publication → correction → withdrawal → tombstone.
- Demonstrate direct publication, duplicate reviewer, missing anchor and public-canary failures.

## T−24 to T−2

- Give the package to an evaluator who did not prepare it.
- Require clean-environment checksum verification and written commands.
- Run all positive and negative cases and classify reproduced/partial/not reproduced.
- Invite only 3–5 evaluators; participation is not endorsement, adoption or traction.
- Test the complaint contact and immediate access-disable procedure.

## T0

Run a 45–60 minute invitation-only session using synthetic data. Lead with limitations. Observe task completion instead of selling. Do not publish participant names, logos or quotes without permission.

## Learning metrics

- unassisted synthetic workflow completion;
- time to first valid anchored candidate;
- second-review reconstruction time;
- predefined boundary-test pass rate;
- forbidden disclosure count (must be zero);
- dated commitment to a second evaluation;
- complaint acknowledgement and containment time.

## Stop conditions

Any real person/data; private field or object key in anonymous output; publication bypass; same-person dual review; mutable audit; checksum mismatch; exposed credential; kill-switch failure; or public claim of production readiness/100x traction.
